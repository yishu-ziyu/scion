# 009 — Tabbit 差距台账与缩差顺序

## 产品判断

**持节的最终形态是 Chrome 插件；对标 Tabbit 的能力与体验，不对标它的浏览器外壳。**

持节今天是一个能跑可验证任务的 Chrome 侧栏原型；Tabbit 已经把搜索、对话、任务、上下文、Skill 与记忆做成完整工作入口。当前最大的差距不是界面，也不是少几个按钮，而是：**持节还没有证明自己能稳定完成足够多的真实复杂工作。**

因此顺序是：先把任务能力和可信度做实，再补上下文、结果交付、复用与记忆，最后把插件打磨成完整工作入口。**插件就是终局，不重新写浏览器。**

## 当前差距

| 维度 | Tabbit 当前公开能力 | 持节当前事实 | 差距 | 下一项证据 |
|---|---|---|---|---|
| 真实复杂任务 | 公开展示跨网页、填表、研究、整理与工作流任务；官方案例页列出 39 类案例 | 已有任务环、页操作、批准、回执；正式产品证据仍集中在少量 fixture 与 Slice A | **关键**：能力面与真实站成功证据都太窄 | 飞书真实填写—批准—单次提交；随后跑“多源读取→结构化结果”内部评测 |
| 可信执行 | 公开披露 Agent 任务成功率 91.8%，但评测协议不可审计 | fixture 已验证；真实飞书/B 站尚未正式验收；动态页面旧目标已加执行前重检并通过 P4 独立验收，尚缺真站证据 | **关键**：现在不能诚实宣称真实工作可托付 | 真实站逐次留页面证据；0 假完成、0 未批提交 |
| 浏览器上下文 | 可引用标签页、标签组、收藏、本地文件、选中文字与截图 | 主要依赖当前页 DOM/视觉；多标签动作存在，但缺少产品级上下文选择与证据 | **高**：用户还不能把“我正在看的全部材料”直接交给 Agent | 最小上下文托盘：当前标签组 + 本地文件 + 选区，进入同一 Task |
| 结果交付 | 可把研究与操作结果沉淀到文档或工作流；媒体报道还提到常见办公文件输出 | 当前完成回执偏运行证据，不是可直接使用的工作成果 | **高**：做完网页动作，不等于把成果交给用户 | 结构化结果页 + Markdown/PDF；外部写入仍走一次批准 |
| 复用与记忆 | 任务可保存为妙招；1.0 公开加入记忆；全文收藏形成知识面 | 本地 Skill 保存与参数化重跑已有实现；跨会话记忆与知识入口未做 | **高**：每次仍像从零开始 | 先把真实成功任务保存并重跑，再做可见、可删、可控的记忆 |
| 插件完整度 | Tabbit 用统一入口承接搜索、对话、任务与上下文 | 侧栏已有任务面，但上下文、结果和 Skill 仍分散 | **高**：插件还不是完整工作入口 | 在侧栏统一“交代任务—给材料—看进度—收结果—保存复用”，复用 Chrome 的标签、收藏与登录态 |
| 模型与生态 | 多模型切换/对比、妙招生态 | 默认 MiniMax-M3；无模型货架与市场 | **中**：不是当前价值瓶颈 | 任务与上下文成熟后再判断，不用功能数量追赶 |

## 缩差顺序

### 现在：把一件真实工作做到可托付

先关掉当前 M3 的飞书真实旅程：自动填写，提交前只问一次，批准后只提交一次，并看到页面成功结果。它直接回答“能不能把一件工作交给持节”。

动态页面动作强绑定已完成本地实现，并通过 P4 独立验收：页面变化后旧目标不执行，稳定目标仍执行，失败交回任务环有限重试。它仍需随真实站旅程一起验证，不能只凭单测宣布差距关闭。

### 接着：从“操作网页”升级为“交付工作成果”

用内部评测任务验证一条完整链路：读取多个公开来源 → 整理重点 → 生成可用文件，或在批准后写入目标网站。**评测集是内部质量工具，不是产品菜单。** 用户看到的产品能力只是“把复杂网页工作交给它，并收到结果”。

### 然后：扩大上下文并让成功可复用

让一个 Task 能接收标签组、本地文件和选区；把成功任务保存为可参数化 Skill；再加入用户可查看、修改和删除的跨会话记忆。

### 最后：让插件成为完整工作入口

当任务、上下文和结果交付已经稳定，把它们收进一个统一侧栏入口，并深度调用 Chrome 已有的标签、收藏、下载与登录态。持节不重写浏览器，只把插件做得足够强大。

## 当前第一项

**P0 决定：第一项缩差是把“真实网页填写并安全提交”做成正式可用证据。插件形态已锁定，不再把自研浏览器列入路线。**

完成标准沿用当前 G3：真实飞书目标、日常登录态、MiniMax-M3；批准前 0 提交，批准后恰好 1 次提交，页面成功态才能完成。缺 Owner 可写地址或登录态时，状态必须写“等待”，不得用 fixture 冒充。

每完成一项，只更新三件事：持节当前事实、差距状态、验收证据路径。没有真实证据，不把“已实现”写成“已缩小”。
