## 3. 核心体验（用户可见）

以 **产品官网 + 官方入门指南** 为主（等级 A）。

### 3.1 统一入口

官方入门指南：`https://www.tabbit.com/guide/usage`

| 能力 | 用户可见行为 | 等级 |
|---|---|---|
| 首页同一输入框 | 访问网页 / 传统搜索 / AI 对话 / Agent 任务 / 妙招 | A |
| 任务模式 | 「更多模式」→「任务」：描述复杂需求，自主执行 | A |
| 对话唤起 | 新标签页对话；右上角 **Chat** 侧栏；划词 / 截图提问 | A |
| 上下文引用 | 输入 `@`：当前标签页、标签组、收藏夹、本地文件 | A |
| 妙招 | `/` 或灯泡：常用提示词 / 脚本 / 任务一键复用；任务完成后可「保存为妙招」 | A |
| 脚本改页 | 自然语言生成脚本改造网页（如去广告）；刷新可取消 | A |
| 标签 | 垂直标签栏；「智能标签整理」；用户区与 Agent 工作区区隔（媒体亦强调） | A / C |
| 收藏 | **全文收藏**（不只 URL）；悬停 AI 摘要；划词/图片可收藏 | A |
| 多模型 | 官网列出可切换国内领先模型（含 MiniMax M3、DeepSeek、Kimi、GLM、Doubao、Qwen、LongCat 等，版本号以官网为准） | A |

官网首页功能叙事（A）：

- **添加引用**：文字 / 截图 / 网页 / 文件作上下文  
- **更多模式**：打开页面、填表、执行操作（自动化重复任务）  
- **任务自主执行** 演示：如「智能客服」理解规范并批量回复（演示视频声明与第三方品牌无隶属）  
- **妙招**、**一句话生成脚本**、**标签智能分组**  
- **隐私承诺** 文案：「主动权始终在你」；数据归属用户（政策细则以产品内隐私页为准，本调研未深读协议全文）

### 3.2 任务 / 人机协作边界（官方能钉住的）

| 陈述 | 来源 | 等级 | 备注 |
|---|---|---|---|
| 购物类任务：自动筛选商品并**创建订单，等待用户最终确认** | 入门指南「任务」典型场景 | A | 这是少数明确的「外部提交前等人」公开句 |
| Agent 可操作网页、跨网页/跨软件复杂任务 | 美团新闻 | B | 「跨软件」无公开技术定义 |
| 云端执行 Agent 编排与代码；本地目录按授权访问 | 36氪/智东西 | C | 未在美团新闻正文出现 |

**未在 A/B 源钉死：**

- 通用「external commit」审批模型（提交/支付/发帖）是否一律停等  
- 「完成」是否强制页面证据，还是模型 `done` 即可  
- 失败分类、重试、过程干预的完整 UX  

→ 这些对 scion 是**产品设计决策**，不能从 Tabbit 公开材料直接抄成事实。

## 4. 准确率口径（严格分层）

### 4.1 主标尺：Agent 任务成功率 53.1% → 91.8%

| 项 | 内容 |
|---|---|
| 数字 | 公测初（约 2026-03）**53.1%** → 1.0 时 **91.8%** |
| 命名 | 「Agent（智能体）任务成功率」 |
| 出处 | [美团新闻中心 · 2026-06-09](https://www.meituan.com/news/NN260612179003187)，正文转述负责人刘炯；页面标注来源「经济参考网」 |
| 等级 | **B**（挂在企业官网，可复核链接；非独立学术论文；分母协议未公开） |
| 示例场景 | HR：从筛选简历到生成 PPT「一句话完成」（叙述性例子，不是评测协议） |
| **不可写进验收的未知** | 任务包构成、是否含跨软件、人工复核标准、是否允许部分成功、模型版本、是否含旗舰独占 |

**对 scion 的含义（分析，非 Tabbit 原文）：**  
003 以 **91.8%** 为对外对齐水平是合理的「对方披露主指标」；但必须用**我方冻结旅程 + verified 定义**审计，**禁止**暗示已复现对方内部 Benchmark。

### 4.2 自研 Benchmark：网页操作 61%→70%；总分 65.0→94.3

| 项 | 内容 |
|---|---|
| 数字 | 网页操作成功率 V0.23 **61%** → V1.0 **70%**；总评分 **65.0 → 94.3** |
| 出处 | [36氪转载智东西 · 2026-06-10](https://36kr.com/p/3845982766680581)，写「在美团自己的 Benchmark 中」 |
| 等级 | **C**（媒体报道发布会/口述；**未**出现在本次抓到的美团新闻正文） |
| 额外媒体句 | 「在线文档、多维表格处理类任务成功率提升 22%」（相对提升，无绝对分母） |

**对 scion：** 003 把 **70%** 作「网页操作底线」有媒体依据，但证据弱于 91.8% 的官网新闻；对外宣称时须写「媒体转述对方自研 bench，非我方复测」。

## 8. 给 scion 的事实边界（P1 建议，非 Captain 决策）

1. 对外对齐句只引用：**Agent 任务成功率 53.1%→91.8%（美团新闻 2026-06-09）**；**网页操作 70% / 总分 94.3** 必须标注媒体转述。  
2. 不得声称「我们复现了 Tabbit Benchmark」。  
3. 竞品是 **原生整浏览器 + 多模型 + 妙招生态 + 记忆**；scion 竞争点应落在 **日常 Chrome 登录态上的任务环正确性**，不是功能面 parity。  
4. Tabbit 已公开「任务模式 + 页操作 + 保存为妙招」；scion 008 的 Slice 顺序与此同构但壳不同。  
5. 准确率数字 **不能替代** Owner 登录态下的 G3/G4 实测。
