# 003 — 北极星（唯一最终目标）

## 一句话

在用户**日常 Chrome 登录态**里，用自然语言委托多步网页操作；Agent **持续做完**、**关键处一次批准**、**只有页面证据才能算完成**；**中等模型（默认 MiniMax-M3）**即可达到可用质量。扩展壳与 Task 契约自建，**Nano 原版 Agent Core 不是终局**。

## 完成时用户可见的样子

1. 侧栏只见「你 / 助手」、目标、状态、下一步；不见 Planner/Navigator/step_failed。
2. 飞书类表单：填完 → 提交前停住等批准 → 批准后只提交一次 → 成功文案/状态出现 → 完成回执。
3. B 站类媒体：播放后说「暂停这个」绑同一对象 → 页面真暂停 → 新回执。
4. 成功任务可存本地 Skill，换输入、页面小改后仍能重跑（语义模板，不是旧索引回放）。
5. 未批准绝不产生外部提交；模型说 done 但证据不足时界面**不得**显示已完成。

## 完成定义（可证伪闸门）

全部满足才算「这个产品阶段完成」，否则不算：

| # | 闸门 | 标准 |
|---|---|---|
| G1 | 本地 fixture 表单 | 连续 10/10：批准前 0 提交、批准后 1 提交、有成功证据、0 假完成 |
| G2 | 本地 fixture 媒体 | 连续 10/10：后续暂停绑原媒体、paused 可证、0 假完成 |
| G3 | 飞书真实旅程 | 固定协议 ≥8/10 verified；0 未批提交；0 假完成 |
| G4 | B 站真实旅程 | 固定协议 ≥8/10；暂停绑定 有效 attempt 内满；0 假完成 |
| G5 | 中等模型 | G1–G4 的正式分母必须用 MiniMax-M3（或同级），禁止只用旗舰刷分 |
| G6 | 核与壳 | 生产路径执行核为可替换适配器；Nano Planner 环可删或已降级；L4（Task/审批/回执/Skill）仍是权威 |
| G7 | 隐私 | 非用户聊天存储中无表单值/凭证/完整正文；无 replay 原始动作键 |

依据：`product/001` PRD、`decisions/001` 扩展载体、`decisions/002` 质量优先可换核。

## 明确不是目标

- 追 star、追 demo、追「更聪明的模型」。
- 默认云浏览器 / 放弃日常登录态。
- 在 Nano Core 上无限堆功能当护城河。
- 只做 UI 改名或只做调研文档而无闸门数字。

## 长程阶段（按序，不并行换题）

```text
M1  P1 控制层 + 薄审批/完成  — fixture G1/G2 稳定 10/10（中等模型）
M2  生产换核设计与接入        — Stagehand/CDP（或等价）接到 TaskManager；Nano 环可拔
M3  真实站点 G3/G4            — 飞书 + B 站固定协议
M4  Skill + 中断/隐私回归     — PRD 交付顺序后半；G6/G7
M5  发布闸门总验收            — 报告 + design 标 current
```

当前进度锚点（2026-07-15）：

- M1：**进行中**。表单/媒体 fixture 各 1 次 PASS；**尚未** 10/10 稳定。
- M2–M5：未开始。

## 会话纪律（防目标模糊）

1. 每一会话只推进**当前 M***，结束时写清：闸门差几项、下一会话只做什么。
2. 禁止无闸门编号的「顺便优化」。
3. 安全缺陷（未批提交、假完成、隐私泄露）可插队热修，但须标回 G 编号。
4. 换题（例如只改 UI）须先改写本文「当前 M」并经 Owner 一句话确认。
