# 003 — 能力天花板 A→C、双声口与私有化复利

## 状态

Owner 口头定案（2026-07-23）。与 `001` 并存：`001` 管 **载体入口**（日常 Chrome + 扩展）；本文管 **能力天花板与演进方向**。

## 决策

### 1. 能力路径：现在 A，终局偏 C，禁止停在「A 就够了」

| 代号 | 含义 | 态度 |
|------|------|------|
| **A** | 扩展本体即可交付：主 Chrome 附着 + observe/act 环 + 中等模型，固定任务集 verified 高 | **当前必须做到**；Phase 1 验收面 |
| **B**（工程手段） | 扩展为脸与任务壳，CDP / Native host / offscreen 等为肌肉 | **允许且鼓励**，当 A 触到 MV3 天花板时接入；对用户仍是「持节插件」 |
| **C** | 能力与控制深度逼近原生浏览器内建 Agent 框架（Tabbit 级附着/权限/ harness 手感） | **方向性终局**；越接近越有未来复利；**不**等于立刻 fork Chromium |

规则：

1. **现在**：优先把 A 做满（会走路）。A 不及格不得用 C 叙事冲业绩。
2. **不自满**：A 达标只是地板。架构预留向 C 加深的接缝（可换执行核、可挂 companion、感知/动作/环契约稳定）。
3. **逼近 C 的方式**：先插件增强（offscreen、Native Messaging、可选 CDP 附着主 Chrome），**默认不**要求用户换浏览器。若将来证据证明扩展生命周期永远达不到目标，再由 Owner 重开 `001`（独立浏览器），不在此文静默改壳。
4. **复利判断**：更接近原生级控制与持续私有上下文 = 更强长期壁垒；功能清单堆砌不算复利。

### 2. 双声口（工程 vs 产品）

| 场合 | 允许 |
|------|------|
| 工程 / 设计 / ADR / 评测 / Agent 内部 | 极致精确：observe-act、stateId、TSR、failure_class、ExecutorDriver、attach_mode… |
| 产品展示 / 侧栏文案 / 对外一句话 / 完成回执给用户看的句子 | **不轴**：人话目标、步骤、结果；不把内核术语甩给用户 |

「不轴」不是模糊：对用户仍要可验证完成；只是 **说法** 像靠谱助理，不像调试台。

### 3. 私有化与「越来越懂用户」

终局产品感：持节是 **私有的、持续理解该用户** 的浏览器行动插件，不是一次性脚本工具。

- Phase 1：可靠操作（parity）。Memory **只预留接口**，不假实现人格。
- Phase 2+：Preference / Workflow / Procedural 等本地私有记忆与飞轮（见 `product/011`），数据默认本机、不默认上传当云脑。
- 「懂用户」建立在 **可验证任务历史 + 本地偏好/技能** 上，不是空喊记忆架构。

## 与 001 / 002 的关系

| 文档 | 管什么 |
|------|--------|
| `001` | 载体：扩展进日常 Chrome；不默认造浏览器、不默认云浏览器 |
| `002` | 质量：执行核可换；不保护 Nano 沉没成本 |
| `003`（本文） | 天花板：A 现在、C 方向；双声口；私有化复利 |

`001` 中「最终产品形态 = 扩展」读作：**用户入口与默认分发形态**。  
`003` 允许在扩展形态内把 **能力深度** 推到接近原生框架；若能力深度最终要求换壳，走 Owner 重开 `001`。

## 后果

- 路线图与 bake-off：先 A 的 TSR；companion / 深度附着标为 **A→C 增强**，不标「已经 C」。
- UI：任务步骤、完成、失败原因用产品汉语；`failure_class` 仅日志/评测/高级诊断。
- 下载、媒体、抓取等：挂在同一可执行框架上；深度能力可走 helper，产品仍说「帮你处理好了 / 这类内容暂时下不了」而不是抛 DRM 缩写甩锅（内部仍记 `drm_blocked`）。
- 规划与代码注释可用极客词汇；PRD 对用户可见章节保持人话。

## 非目标

- 本决策不批准现在开工 fork Chromium。
- 本决策不把 Phase 2 记忆实现塞进 Phase 1 sprint。
- 本决策不取消 verified completion 或质量优先。
