Convergence order

缩差顺序已经定了

先证明价值，再扩大材料，再积累复用，最后改变产品形态。

把真实网页工作做到可托付

先关飞书真实旅程，同时补动态页面动作强绑定。不是追求演示，而是证明不会误点、不会假完成。

用户结果：填写正确、批准一次、只提交一次、看到成功

从网页操作升级到成果交付

读取多个公开来源，整理重点，输出结构化文件；或在批准后写入目标网站。

用户结果：拿到能直接使用的结果，而不是一串执行日志

扩大上下文，让成功可以复用

标签组、文件、选区进入同一 Task；成功任务保存为 Skill，再加入可控记忆。

用户结果：它知道材料，也不必每次从零教

让插件成为完整工作入口

统一任务、材料、进度、结果与 Skill，并深度使用 Chrome 已有标签、收藏、下载和登录态。

用户结果：不用换浏览器，也能把复杂工作交给 Agent
!
评测只在团队内部

“从公开账号读取内容 → 判断重要性 → 写飞书或生成 PDF/邮件”是用来检验整条链路的任务，不是要做成产品里的一个按钮。产品能力只有一句话：把复杂网页工作交给它，并收到结果。
